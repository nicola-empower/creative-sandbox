Since you are using **Astro**, this is perfect. Astro is built for this kind of thing because it lets you create isolated pages easily without complex routing setups.

Since this is your "Sandbox" project (separate from the main site), you want a structure that lets you spin up **The Disruptor**, **The Titan**, etc., quickly, record them, and move on.

Here is the cleanest file and folder set up for your **Astro Vibe Lab**.

### **1. The File Structure (Visual Tree)**

You want to keep each "Vibe" isolated so their CSS doesn't bleed into each other.

```text
/my-astro-sandbox
├── public/
│   └── images/              <-- Put your raw background images here
├── src/
│   ├── layouts/
│   │   └── BlankLayout.astro  <-- VITAL: A layout with NO header/footer
│   ├── pages/
│   │   └── vibes/           <-- Create this folder
│   │       ├── disruptor.astro
│   │       ├── titan.astro
│   │       └── curator.astro
│   └── styles/              <-- Optional: if you want shared reset.css
└── astro.config.mjs
```

### **2. The "Blank" Layout**

**Why you need this:** Your default Astro layout probably has a Navbar and Footer. You don't want those in your "Vibe" video recordings. You want a clean slate.

Create `src/layouts/BlankLayout.astro`:

```astro
---
// src/layouts/BlankLayout.astro
const { title } = Astro.props;
---
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>{title}</title>
    <style is:global>
      *, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }
      body { width: 100%; height: 100vh; overflow-x: hidden; }
    </style>
  </head>
  <body>
    <slot />
  </body>
</html>
```

### **3. The Vibe Page (e.g., The Disruptor)**

Now, create the specific page. This is where you paste the Hex codes and fonts we discussed.

1.  **Create** `src/layouts/BlankLayout.astro` (The clean slate).
2.  **Create** `src/pages/vibes/` folder.

4.  **Run** `npm run dev`.
